Alcove Branching Strategy
To keep our main branch stable and organized, we follow these three rules:

1. Branch Naming
   Every branch must start with your name followed by what you are working on.

Format: yourname/feature-name

Examples: - alexa/landing-page

            alexa/artist-photos

2. Commit Messages
   Keep your commit messages clear and descriptive so the rest of the team knows what you changed.

Good: "added the navigation bar links" or "fixed the image alignment on mobile"

Avoid: "fixed stuff" or "update"

3. Merging to Main

No Direct Pushes: Never push your code directly to the main branch.

Pull Requests: Always create a Pull Request (PR) on GitHub to merge your branch into main.

Approval: Get at least one teammate to look over your PR and give it a "thumbs up" before you merge it.
